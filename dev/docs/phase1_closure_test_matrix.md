# 第一阶段闭环测试矩阵

本文档用于对照《Windows 兼容 Claude / Cursor / Codex 的 MCP 设计文档 v1》中第一阶段交付目标，记录“小模块 / 小闭环 / 最终阶段”的测试覆盖情况与后续联调路径。

## 第一阶段交付范围

- MCP 基础接入
- 窗口状态读取
- 截图与 UIA 快照
- 点击、输入、快捷键
- 简单任务编排
- 安全确认

## 测试分层定义

- **小模块**：单个函数、单个工具、单个 schema、单个执行路径。
- **小闭环**：一个完整可运行链路，例如“读取状态 -> 执行动作 -> 验证返回”。
- **最终阶段**：第一阶段全部能力在真实 Windows 桌面与真实后端上的联调验证。

## 后续联调路径

### 1. 真实后端联调

目标：把当前 facade / stub 的执行路径切换到真实 Windows-MCP 后端，至少完成以下验证：

- 窗口枚举与激活
- 鼠标点击与键盘输入
- 快捷键发送
- 窗口状态读取
- 截图 / UIA 快照输出

### 2. 真实桌面环境验证

目标：在真实 Windows 桌面环境中验证第一阶段接口返回值与实际桌面状态一致。

建议验证点：

- 截图是否能反映当前前台窗口。
- UIA 快照是否能覆盖当前焦点控件。
- 点击与输入后，UIA 树是否出现对应变化。
- 快捷键后，窗口焦点或应用状态是否改变。

### 3. 第一阶段最终闭环

目标：形成可交付的第一阶段系统闭环。

建议闭环顺序：

1. MCP 启动。
2. 读取窗口与桌面状态。
3. 拉取截图 / UIA 快照。
4. 执行点击 / 输入 / 快捷键。
5. 再次读取状态。
6. 对比前后状态，确认动作生效。
7. 输出统一结构化结果。

## 交付判断标准

当满足以下条件时，可认为第一阶段达到可交付状态：

- 所有第一阶段工具均可注册并可调用。
- 关键工具返回值满足统一 schema。
- 真实后端联调通过。
- 真实桌面环境验证通过。
- 端到端闭环测试通过。
