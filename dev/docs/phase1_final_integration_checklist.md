# 第一阶段最终联调清单

本文档用于第一阶段交付前的最终联调检查。目标是把“能跑的 stub / facade”提升为“真实 Windows 桌面环境下可验证的闭环系统”。

## 联调目标

第一阶段最终联调只验证以下能力：

- MCP 基础接入
- 窗口状态读取
- 截图与 UIA 快照
- 点击、输入、快捷键
- 简单任务编排
- 安全确认

## 联调前提

- 已安装并可运行真实 Windows-MCP 后端。
- 测试机器可访问真实桌面环境。
- 目标应用已准备好可观察、可点击、可输入的窗口。
- 安全闸默认开启。

## 最终联调清单

### 1. MCP 基础接入

- [ ] 启动 `desktop-agent-dev`。
- [ ] 确认 MCP Server 正常注册工具。
- [ ] 确认客户端可看到完整工具列表。
- [ ] 确认工具 metadata 可被读取。

验收标准：

- 服务可启动。
- 工具列表完整。
- 参数 schema 与返回 schema 可读。

### 2. 窗口状态读取

- [ ] 读取当前活动窗口。
- [ ] 读取窗口列表。
- [ ] 读取窗口位置与尺寸。
- [ ] 读取窗口焦点信息。

验收标准：

- 读取结果与真实桌面状态一致。
- 活动窗口切换后结果会变化。

### 3. 截图与 UIA 快照

- [ ] 采集当前桌面截图。
- [ ] 采集当前 UIA 树快照。
- [ ] 验证截图对应前台窗口。
- [ ] 验证 UIA 节点能覆盖当前可交互区域。

验收标准：

- 截图内容正确。
- UIA 快照结构完整。
- 快照变化能反映真实界面变化。

### 4. 点击、输入、快捷键

- [ ] 对目标控件执行点击。
- [ ] 输入文本到当前焦点。
- [ ] 执行快捷键组合。
- [ ] 验证动作后 UI 状态发生变化。

验收标准：

- 点击后焦点或控件状态变化。
- 输入后文本出现在目标位置。
- 快捷键后窗口或应用行为符合预期。

### 5. 简单任务编排

- [ ] 输入一个简单目标，生成计划。
- [ ] 读取任务状态。
- [ ] 走通“计划 -> 执行入口 -> 状态确认”的链路。

验收标准：

- 计划可生成。
- 状态可读取。
- 任务步骤格式统一。

### 6. 安全确认

- [ ] 触发需要安全确认的动作。
- [ ] 验证默认阻断是否生效。
- [ ] 验证确认通过后才可继续执行。

验收标准：

- 未确认时阻断。
- 确认后放行。
- 审计信息可追踪。

## 联调顺序建议

建议按下面顺序执行，降低排错成本：

1. MCP 基础接入
2. 窗口状态读取
3. 截图与 UIA 快照
4. 点击、输入、快捷键
5. 简单任务编排
6. 安全确认
7. 端到端完整闭环

## 端到端最终闭环

最终联调必须验证以下完整链路：

1. 接收目标。
2. 生成任务计划。
3. 读取桌面和窗口状态。
4. 采集截图和 UIA 快照。
5. 执行点击 / 输入 / 快捷键。
6. 再次读取状态。
7. 对比前后结果。
8. 输出结构化成功结果。

## 当前状态判断

- **stub / facade 链路**：已可用于开发测试。
- **真实后端联调**：仍需执行。
- **真实桌面验证**：仍需执行。
- **第一阶段最终交付**：需在真实后端联调完成后确认。

## 完成标准

当以下条件全部满足时，认为第一阶段最终联调完成：

- 工具注册完整。
- 真实后端可用。
- 真实桌面可验证。
- 第一阶段工具全部通过集成测试。
- 端到端闭环无阻断。
